Git基本命令整理

Mar 13, 2017

Git是一个开源的分布式版本控制系统，可以有效、高速的处理从很小到非常大的项目版本管理。Git 是 Linus Torvalds 为了帮助管理 Linux 内核开发而开发的一个开放源码的版本控制软件。

  1. git –version 查看Git的版本
  2. git config –list 查看Git的配置信息
  3. git config –global user.name “Your Name” 设置用户名
  4. git config –global unset user.name “Your Name” 删除用户名
  5. git config –global user.email “Your Email” 设置用户邮箱
  6. git config –global unset user.email “Your Email” 删除邮箱
  7. git init 设置版本仓库
  8. git add –add 或者 git add . 添加所有更改的文件
  9. git add <file> 添加某一个文件
  10. git commit -m “commit message” 添加更改信息，必须添加，不然报错
  11. git commit –amend -m “commit again” 追加修改
  12. git status 查看git当前状态
  13. git diff <file> 查看file 到底修改了什么内容
  14. git log 查看最近的提交日志信息
  15. git log –pretty=oneline 单行显示提交日志信息
  16. git reflog 查看命令的历史信息
  17. git checkout –<file>利用版本库中的版本替换工作区中的文件。功能有2：撤销文件修改，分两 种情况：撤销工作区中的修改（没有使用git add命令添加到暂存区）撤销暂存区中的修改（添加到了暂存区又做了修改）找回删除的文件工作区中文件误删了，可以通过此命令从版本库中找回
  18. git reset HEAD <file> 撤销add，回到工作区
  19. git rm <file> 删除文件
  20. git remote add origin “Your URL” 将本地版本库关联到github远程库上
  21. git oush -u origin master 第一次推送的时候要加上 -u 参数，可以将本地版本库的master分支与远程库的master分支关联，下次提交就不需要加 -u 参数
  22. ssh-keygen -t rsa -C “Your Email” 创建SSH Key
  23. git checkout -b dev 创建 dev 分支并切换到dev分支，相当于git branch dev 、git checkout dev 两个命令的使用
  24. git branch 查看当前分支
  25. git branch -d dev 删除dev分支
  26. git log –graph –pretty=oneline –abbrev-commit 查看分支合并图
  27. git merge dev 合并指定分支到当前分支
  28. git merge –no-ff -m “merge with no-ff” dev 禁用[Fast forward],也就是保留分支的相关信息
  29. git stash 将工作区现场储存起来，等以后恢复继续工作，通常用于处理更为着急的任务，例如：bug
  30. git stash list 查看保存的工作现场
  31. git stash apply 恢复工作现场
  32. git stash drop 删除stash 内容
  33. git stash pop 恢复的同时直接删除stash 内容
  34. git branch -D dev 强制删除dev分支
  35. git remote 查看远程库的信息
  36. git remote -v 查看远程库的详情信息
  37. git clone “URL” 将远程库克隆到本地，默认只能看到master分支
  38. git checkout -b dev origin/dev 创建远程dev分支到本地
  39. git pull 将远程分支的最新内容抓取下来
  40. git branch –set-upstream dev origin/dev将本地dev分支与远程dev分支之间建立链接。多人协作工作模式首先，可以试图用git push origin branch-name推送自己的修改；如果推送失败，则因为远程分支比你的本地更新，需要先用git pull试图合并（如果git pull提示“no tracking information”，则说明本地分支和远程分支的链接关系没有创建，用命令git branch –set-upstream branch-name origin/branchname）;如果合并有冲突，则解决冲突，并在本地提交；没有冲突或者解决掉冲突后，再用git push origin branch-name推送就能成功
  41. git tag v1.0 给当前分支打上标签
  42. git tag 查看所有的标签
  43. git show v1.0 查看标签的信息
  44. git tag -a v0.1 -m “version 0.1 released” commitID创建带有说明的标签，-a指定标签名，-m指定说明文字
  45. git tag -d v0.1 删除标签v0.1
  46. git push origin v1.0 推送标签1.0到远程
  47. git push origin –tags 推送所有的 标签到远程
  48. git push origin :refs/tags/v0.9 删除远程标签，但是前提是要先在本地删除对应标签
